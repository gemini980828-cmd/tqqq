C안

처음부터 분석 도구처럼 넓게 펼쳐 정보량이 많지만, 구현 범위와 복잡도가 빠르게 커질 수 있음.

A안이 좋은 점

  • 점수순 스캔 흐름이 매우 선명함
  • 이번 세션 목표인 usable surface와 잘 맞음
  • Home / Inbox 최소 계약으로 연결하기 쉬움

C안의 주의점

  • 처음부터 정보량이 많아 집중 포인트가 흐려질 수 있음
  • 이번 세션 범위를 넘어 과설계될 위험이 큼
  • 다른 manager와의 일관성도 더 크게 흔들 수 있음
