# Risk-on Dip-Add Overlay Design

**작성일**: 2026-03-28  
**대상 엔진**: `phase2-best + soft_overheat_buffer(129/123/0.90)`  
**상태**: 제안 / 구현 전 검토 필요

---

## 1. 문제 정의

현재 메인 엔진은 **회복 확인 후 진입형**입니다. 이 구조는 약세 구간 방어에는 강하지만, 이미 risk-on 체제 안에서 발생하는 짧은 급락 뒤 반등을 일부 놓칠 수 있습니다.

이번 설계의 목표는 다음과 같습니다.

- **현금 구간 역추세 진입 없이**
- **기존 risk-on 체제 내부에서만**
- 짧은 눌림목 구간에 한해
- **+5% ~ +10%의 제한적 가산**을 허용해
- CAGR 소폭 개선 가능성을 탐색한다

핵심은 새로운 buy-the-dip 전략을 추가하는 것이 아니라, 기존 최종 엔진 위에 **아주 작은 tactical overlay**를 얹는 것입니다.

---

## 2. 비목표 / 금지 범위

이 설계는 의도적으로 다음을 하지 않습니다.

- **0% 현금 상태에서의 역추세 신규 진입**
- SPY bear 상태에서의 추가 매수
- vol lock 상태에서의 추가 매수
- soft overheat buffer와 동시에 작동하는 상충 로직
- 독립적인 전술 매매 엔진으로의 확장

즉 이 설계는 “떨어지면 산다”가 아니라,  
**“이미 강세 체제일 때만 눌림목을 아주 조금 더 산다”**를 목표로 합니다.

---

## 3. 고려한 접근법

### 접근 1 — 현금 구간 dip buy
- 약세 / 현금 상태에서도 급락 시 일부 진입
- **장점**: 반등 초기 CAGR 개선 여지 큼
- **단점**: SPY 필터·vol lock 철학과 충돌, MDD 악화 위험 큼
- **판단**: 기각

### 접근 2 — risk-on 내부 dip add
- 이미 risk-on인 상태에서만 단기 급락에 반응
- **장점**: 기존 엔진 철학 유지, 개입 범위 제한 가능
- **단점**: 개선 폭이 크지 않을 수 있음
- **판단**: 채택

### 접근 3 — overheat와 병행하는 dip add
- 과열 완충이 켜져 있어도 dip-add 허용
- **장점**: 특정 과열 조정 구간에서 반등 회복 일부 포착 가능
- **단점**: “과열이라 줄인다”와 “눌렸으니 더 산다”가 충돌
- **판단**: 기각

**추천 접근**은 접근 2입니다.

---

## 4. 최종 제안: `risk_on_dip_add_v1`

### 4.1 설계 원칙
- 기존 최종 엔진을 **대체하지 않는다**
- 진입이 아닌 **가산만** 담당한다
- 발동 구간을 엄격히 제한한다
- 짧게 유지하고 빠르게 원상 복귀한다
- 복잡성을 늘리더라도 설명 가능성을 유지한다

### 4.2 위치
- `phase2-best` 코어 계산 후
- `soft_overheat_buffer`와 결합한 최종 target 계산 시점 근처에서
- 별도 overlay로 적용

권장 구현 형태:
- `RiskOnDipAddOverlay`
- `apply_risk_on_dip_add(...)`

---

## 5. 발동 조건

아래를 **모두 만족**할 때만 dip-add를 켭니다.

1. **SPY bull 상태**
2. **vol lock 아님**
3. **reentry lock 아님**
4. **soft_overheat_buffer OFF**
5. **base target >= 90%**
6. **2일 누적 TQQQ 수익률 <= -10%**

### 5.1 dip 강도별 가산 규칙
- `2일 누적 수익률 <= -10%` → `+5%`
- `2일 누적 수익률 <= -14%` → `+10%`

### 5.2 상단 제한
- 최종 target은 **100% 초과 금지**

즉 base target이 97%이고, 심한 눌림목이 와도 최대 100%까지만 허용합니다.

---

## 6. 해제 규칙

dip-add는 **짧게만** 유지합니다.

### 해제 조건
- **최대 2거래일 유지**
- 또는 그 전에
  - **2일 누적 수익률 >= 0%** 이면 즉시 해제

즉 “짧게 더 담고, 빠르게 원래 엔진으로 복귀”하는 구조입니다.

---

## 7. 상태 예시

### 예시 A
- base target = 90%
- buffer = OFF
- 2일 누적 수익률 = -10.8%

→ 최종 target = **95%**

### 예시 B
- base target = 97%
- buffer = OFF
- 2일 누적 수익률 = -14.6%

→ 최종 target = **100%**

### 예시 C
- base target = 97%
- buffer = ON
- 2일 누적 수익률 = -15%

→ **dip-add 금지**, 기존 엔진 최종 target 유지

---

## 8. 기대 효과

### 기대되는 개선 포인트
- 이미 강세 체제인 구간의 눌림목 반등 일부를 더 포착
- 회복 확인형 코어의 다소 늦은 반응을 부분 보완
- CAGR 소폭 개선 가능성

### 기대하지 않는 것
- 약세장 바닥 예측
- 캐치-더-폴링-나이프
- 대규모 CAGR 점프

이 설계는 **작은 개선 가능성**을 탐색하는 것이지, 전략 성격을 바꾸려는 시도가 아닙니다.

---

## 9. 리스크

### 9.1 약세 전환 초입 오판
2일 급락이 단순 눌림목이 아니라 더 큰 하락의 시작일 수 있습니다.

### 9.2 OOS 취약성
특정 반등 구간 몇 번이 백테스트 결과를 과하게 끌어올릴 수 있습니다.

### 9.3 복잡성 증가
현재 엔진은 이미 충분히 정리된 상태이므로, 추가 로직은 반드시 **작고 설명 가능**해야 합니다.

---

## 10. 검증 기준

이 overlay는 구현보다 **검증 기준 선고정**이 더 중요합니다.

### 필수 지표
- **after-tax CAGR**
- **pretax MDD**
- **OOS retention**
- 발동 일수 / 추가 가산 일수 / turnover

### 통과 기준
- after-tax CAGR: 현재 최종 엔진 대비 **개선 또는 동급**
- pretax MDD: 의미 있는 악화가 없어야 함
- OOS retention: **0.70 이상 유지**

### 즉시 기각 기준
- **OOS retention < 0.70**
- MDD가 명확히 악화
- CAGR 개선은 미미한데 turnover/복잡성만 증가
- 특정 스트레스 구간에서만 과도하게 좋아짐

## 12. 최종 권고

`risk_on_dip_add_v1`는 **실험해볼 가치가 있는 보수적 후보**입니다.  
다만 이 기능은 현재 최종 엔진을 대체하는 것이 아니라,  
**현재 최종 엔진 위에 얹는 소형 실험 overlay**로 취급해야 합니다.

즉 이 설계의 올바른 질문은:

> “buy-the-dip를 넣을까?”가 아니라,  
> **“risk-on 내부 눌림목에 한해 5~10% 단기 가산이 실전적으로 의미 있는가?”**

입니다.

현재 판단으로는:
- **설계 자체는 승인 가능**
- **구현은 가능**
- 하지만 **채택은 반드시 검증 결과를 본 뒤 결정**해야 합니다.
