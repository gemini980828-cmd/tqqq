### 래퍼런스에서 추출한 4가지 공통 원칙

#### 1) Robinhood / Fidelity — "행동 가능한 정보가 먼저"

- 차트/퍼포먼스가 먼저 눈에 들어옴
- 주문/투자 행동으로 바로 이어짐
- **시사점**: "오늘 해야 할 행동 1개"와 "현재 상태 1개"만 먼저. 설명 카드가 앞서면 안 됨

#### 2) Wealthfront / Betterment / Empower — "내 상태를 먼저 안정적으로"

- 계좌/현금/전체 그림이 잘 보임
- 설명은 짧고, 디테일은 접음
- **시사점**: 현재 보유상황은 더 앞에 와야 하고, 주문 흐름과 한 세트로 보여야 함

#### 3) Schwab / Vanguard — "신뢰감 있는 요약 + 필요한 동작"

- 보수적이고 정돈된 구조
- 보조 기능은 시각적으로 덜 튐
- **시사점**: 간결/상세 토글은 더 작아지고, 메인 정보보다 덜 튀어야 함

#### 4) Bloomberg / Morningstar / IBKR — "기본층 / 확장층이 분명"

- 정보량 많아도 기본 화면은 요약, 세부는 drill-down
- 이벤트/경고/관찰 포인트가 시각적으로 잘 구분됨
- **시사점**: AI 운영매니저도 접힌 도움말이 아니라, 기본 운영 브리프 + 확장 질문 구조여야 함

## Part 2. 핵심 문제 진단 — 합의된 사항

### ✅ 합의 ❶: AI 운영매니저가 아직 약하다

**현재**: 최하단 설명 섹션. "헷갈릴 때만 설명 보기" 느낌.  
**문제**: 운영비서/운영매니저가 아니라 **도움말 박스**처럼 보임. 톤도 "AI가 한 번 더 풀어드립니다" 같은 교사→학생 스타일.

### ✅ 합의 ❷: 주문 카드가 중복 설명처럼 느껴진다

**현재**: "오늘 할 일"에서 12주 매수 → 아래 주문 입력값에서도 12주 매수.  
**문제**: 설명 강화가 아니라 **시각적 중복**으로 느껴짐.

### ✅ 합의 ❸: 실행 영역의 흐름이 끊겨 있다

**현재**: 주문 실행 → (다른 섹션들) → 체크리스트 → (다른 섹션들) → 주문 후 기록  
**문제**: 확인하고 나서 주문해야 하는데, 주문을 먼저 보고 확인은 나중에 봄.

### ✅ 합의 ❹: 행동 없는 날의 경험이 없다

**현재**: 액션 있든 없든 거의 비슷한 밀도/길이.  
**문제**: 행동 없는 날에는 훨씬 짧고 차분해야 하는데, 동일한 밀도로 보여줌.

## Part 4. 남은 핵심 3개 — 현재 기준 실행 우선순위

### 🥇 1순위: AI 운영매니저를 상단 고정 운영 브리프로 승격

**현재**: 하단 설명 박스  
**목표**: 우측 상단 고정 카드

#### 제안 구조

```
┌─────────────────────────────────┐
│ AI 운영매니저 브리프             │
│                                  │
│ 📋 오늘 판단                     │
│   목표 95%에 맞추려면            │
│   12주 매수가 맞습니다.          │
│                                  │
│ ⚠️ 지금 경고                     │
│   기준가와 오늘 가격 괴리가      │
│   없는지 확인하세요.             │
│                                  │
│ ➡️ 다음 행동                     │
│   삼성증권 주문 입력 후          │
│   체결 여부를 아래에 기록하세요. │
│                                  │
│ ┌──────┐ ┌──────┐ ┌──────┐      │
│ │Q1    │ │Q2    │ │Q3    │      │
│ └──────┘ └──────┘ └──────┘      │
│   quick ask 3개                  │
└─────────────────────────────────┘
```

#### 톤 변경

| 현재 (도움말 톤)             | 제안 (운영 비서 톤)              |
| ---------------------------- | -------------------------------- |
| "AI가 한 번 더 풀어드립니다" | **"AI 운영매니저 브리프"**       |
| "현재 비중이 86.63%이고..."  | **"판단: 12주 매수가 맞습니다"** |
| 길게 풀어쓴 설명 텍스트      | 판단 1줄 + 경고 1줄 + 행동 1줄   |

#### 래퍼런스 근거

- **Bloomberg**: 기본 화면은 요약, 세부는 drill-down. AI도 기본 브리프 + 확장 질문 구조여야 함
- **Schwab**: 검색/도움말을 헤더에 항상 고정. 보조 기능이 어디서든 접근 가능
- **Robinhood**: 핵심 정보가 먼저, 맥락은 나중. AI도 판단이 먼저 보이고 설명은 접혀야 함

---

### 🥈 2순위: 실행 영역을 한 묶음으로 재배치

**현재**: 주문 실행 → (중간에 다른 섹션) → 체크리스트 → (중간) → 주문 후 기록  
**목표**: 아래 4개가 같은 Zone에 연속 배치

```
┌─────────────────────────────────┐
│ ① 현재 보유상황                  │
│    86.63% ━━━━━━━━━○ 95.00%     │
├─────────────────────────────────┤
│ ② 주문 실행 (검산용)             │
│    왜: GAP 8.37%p               │
│    얼마: 972,000원              │
│    기준가: 81,000원             │
├─────────────────────────────────┤
│ ③ 주문 전 확인                   │
│    ☑ 계좌 입력 최신화            │
│    ☑ 기준가 확인                 │
│    ☑ 보유 현금 확인              │
├─────────────────────────────────┤
│ ④ 주문 후 기록                   │
│    체결 수량: ___               │
│    체결가: ___  시각: ___       │
└─────────────────────────────────┘
```

**핵심 원칙**: 이 4개 사이에 다른 섹션이 끼어들면 안 됨.  
사용자의 행동 흐름: **"뭐 갖고 있지?" → "얼마만큼 사지?" → "확인했어?" → "기록하자"**

#### 래퍼런스 근거

- **Robinhood/Fidelity**: 행동 가능한 핵심 정보 → 주문/투자 행동으로 바로 이어짐
- **Wealthfront/Empower**: 현재 내 상태를 먼저 → 그래서 이걸 한다

---

### 🥉 3순위: 액션 없는 날 / 있는 날 레이아웃 분기

**현재**: 모든 날 동일한 밀도/길이  
**목표**: 상태에 따라 페이지 구조 자체가 달라짐

#### Hold (유지) 상태

```
① 상단 브리프 — "오늘 특별한 거 없음. 리스크 게이지 안정."
② 현재 보유상황 — 간결하게
③ 성과 차트
④ 간단 점검 (Risk Gauge만)
```

→ 짧고 차분. **스크롤 ~800px 이내**.

#### Buy/Sell (매수/매도) 상태

```
① 오늘 할 일 — "TQQQ 12주 매수"
② AI 운영매니저 브리프
③ 현재 보유상황
④ 주문 실행 + 체크리스트 + 주문 후 기록 (한 묶음)
⑤ 성과 차트
⑥ 상세 근거 (접힘)
```

→ 실행 중심. **스크롤 ~1200px**.

#### 래퍼런스 근거

- **Robinhood**: 특별한 일 없으면 차트 하나 + 잔고 한 줄이 끝
- **Vanguard**: 행동이 필요 없을 때는 "안정적" 분위기, 필요할 때만 CTA 강조
